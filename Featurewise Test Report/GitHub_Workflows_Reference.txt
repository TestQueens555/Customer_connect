THREE WORKFLOW FILES CREATED
------------------------------
Location: D:\Claude\QA_Projects\CustomerConnect\.github\workflows\

  1. playwright.yml   — Runs on every push to main + every PR
  2. ci.yml           — Runs on every Pull Request (lint + smoke test gate)
  3. regression.yml   — Runs nightly at 12:00 AM UTC + manual on-demand


══════════════════════════════════════════════════════════════
 1. playwright.yml — E2E Tests on Push / PR
══════════════════════════════════════════════════════════════

TRIGGERS:
  - Push to main branch
  - Pull request targeting main
  - Manual (workflow_dispatch)

WHAT IT DOES:
  1. Checkout code
  2. Setup Node.js 18
  3. npm install
  4. npx playwright install chromium --with-deps
  5. Run login tests  →  npx playwright test tests/login.spec.js
  6. Generate Login.xlsx  →  node utils/generate-report.js --feature=Login
  7. Run all tests  →  npx playwright test
  8. Save reference doc  →  node utils/save-reference.js
  9. Upload HTML report artifact  (retained 30 days)
  10. Upload Featurewise Excel reports artifact  (retained 30 days)

ARTIFACTS PRODUCED:
  playwright-html-report-{run_id}      ← HTML test report
  featurewise-test-reports-{run_id}    ← Excel reports folder


══════════════════════════════════════════════════════════════
 2. ci.yml — Pull Request Gate (Lint + Smoke Test)
══════════════════════════════════════════════════════════════

TRIGGERS:
  - Pull request opened/updated targeting main
  - Manual (workflow_dispatch)
  - Called by regression.yml (workflow_call)

JOB 1 — lint-check:
  - Validates all required files exist (pages, tests, utils, config)
  - Validates all npm scripts exist
  - Runs node --check on all JS files (syntax validation)

JOB 2 — smoke-test (runs after lint-check passes):
  - Installs Playwright
  - Runs TC-LOGIN-001 only as a quick smoke check
  - Uploads smoke test report artifact (retained 7 days)

ARTIFACTS PRODUCED:
  smoke-test-report-{run_id}    ← Smoke test result


══════════════════════════════════════════════════════════════
 3. regression.yml — Nightly + On-Demand Regression
══════════════════════════════════════════════════════════════

TRIGGERS:
  - Nightly schedule: 0 0 * * *  (12:00 AM UTC = 5:30 AM IST)
  - Manual with feature selector:
      all / Login / Dashboard / Registration / Profile

PIPELINE:
  Job 1 — pre-checks: reuses ci.yml as gate
  Job 2 — regression (only if pre-checks pass):
    1. Run full test suite
    2. node utils/generate-report.js --feature=Login --regression
    3. node utils/generate-report.js --feature=Dashboard --regression
    4. Auto-generate reports for any other *.spec.js files found
    5. Save reference doc
    6. Upload HTML + Excel artifacts (retained 60 days)

ARTIFACTS PRODUCED:
  playwright-regression-html-{run_id}        ← HTML report
  featurewise-regression-reports-{run_id}    ← Excel reports


══════════════════════════════════════════════════════════════
 HOW TO VIEW ARTIFACTS IN GITHUB
══════════════════════════════════════════════════════════════

1. Go to your GitHub repo
2. Click "Actions" tab
3. Click on any workflow run
4. Scroll to bottom → "Artifacts" section
5. Download the Excel zip → extract → open Login.xlsx etc.


══════════════════════════════════════════════════════════════
 HOW TO TRIGGER MANUAL REGRESSION
══════════════════════════════════════════════════════════════

1. Go to GitHub repo → Actions tab
2. Click "Regression — Nightly & On-Demand"
3. Click "Run workflow"
4. Select feature (Login / Dashboard / all)
5. Click "Run workflow" button
